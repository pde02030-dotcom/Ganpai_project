# AI 음성 응대 앱 — MVP 설계서 v0.2

> **타겟 가게**: 부산 광안리 일본풍 타치노미(스탠딩 바) 선술집 / 생맥주 음식점
> **모델**: 손님 셀프형 (확정) → 2단계에서 직원 통역 모드 추가 = 하이브리드 도달
> **핵심 손님**: 외국인 비중 40% (영어 / 중국어 / 일본어)
> **AI의 핵심 가치**: 직원 노동 대체가 아니라 **언어 장벽 해소**. 외국인 40%에게 영/중/일 1차 응대를 AI가 맡고, 홀 직원은 곤란·복잡 응대를 유연하게 백업.
> **작성**: 김선국 (COO/CTO) · J 협의 필요 항목 별도 표기
> **버전**: v0.7 — FAQ #2 쇼맥 설명 포함 확정. 남은 빈칸 = #5 알레르기(보류) + #7 화장실 비번·와이파이 값. (v0.6: FAQ 변수 실데이터 입력 / v0.5: 내레이션 4개 언어 대본 확정) (v0.3: 실제 응대 흐름 + 카드 선결제 + NFC 반영, FAQ를 "착석 후 Q&A 도우미"로 재정의)
> **매장명**: ganpai 광안2호점 (Sapporo 퍼펙트 푸어링 기법)

## 0. 이 문서의 목적

코드 들어가기 전, **무너지지 않는 1단계 뼈대**를 종이에 먼저 박는다.
세 가지를 확정한다:

1. FAQ 10개 초안 (= 1단 캐시의 원본)
2. 곤란질문 → 직원 호출(에스컬레이션) 케이스
3. 1단(즉답) / 2단(생성) 응답 구조 설계

> **머리는 충분하다. 실행이 인생을 결정한다.**
> 이 문서가 완성되면 → FAQ 변수만 채워서 1단 캐시부터 만든다. 추상 설계는 여기서 끝.

## 1. 사용 시나리오 (손님 셀프형)

```
① 외국인 손님 입장
        │  (홀 직원: "Hi, how many people?" / 손님: 외국어로 인원 답)
        ▼
② 홀 직원이 착석 안내 + 번역 메뉴판 전달 + 태블릿 세팅
        │  (스탠딩바·인원·가격은 이 단계에서 이미 해결 → FAQ 불필요)
        ▼
③ 테이블 NFC 태그로 앱/태블릿 실행  ← AI 도우미 등장 지점
        │
        ▼
④ 머무는 동안 손님이 자기 언어(영/중/일)로 음성 질문  [푸시투토크]
        │
        ▼
⑤ AI가 텍스트 + 음성으로 같은 언어로 응답
        │
   ┌────┴─────┐
   ▼          ▼
FAQ면 즉답    FAQ 밖이면 가게 맥락으로 생성 응답
              │
              ▼
        곤란/민감 질문이면 → "직원 호출" 버튼 + 홀 직원 알림
```

> **AI의 역할 재정의 (v0.3)**: 입장·주문·결제는 홀 직원 + 메뉴판 + 카드 선결제로 처리된다. AI는 **착석 후 머무는 동안의 Q&A 도우미** — 외국인이 직원을 부르기 애매한 자잘한 질문(추천·알레르기·화장실·재주문 등)을 자기 언어로 즉시 해결.

**직원 구조 (v0.2 반영)**
- **푸어링 직원**: 생맥주 따르느라 손이 없음 → 외국어 응대 불가. AI가 이 부담을 대체.
- **홀 직원**: 손도 있고 유연하게 대응 가능 → QR/태블릿 안내, 곤란질문 백업을 충분히 소화. 단, 외국어 능숙 응대는 어려움 → 여기서 AI가 언어 다리 역할.
- 결론: "직원 개입 0"이 아니라 **"AI 1차 응대 + 홀 직원 유연 백업"**.

**핵심 설계 원칙**
- AI가 외국인 손님의 언어·반복 FAQ를 1차로 처리 → 홀 직원은 곤란·복잡 케이스에 집중.
- 곤란질문은 **홀 직원에게 호출** → 손이 있으니 즉시 유연 대응 가능 (에스컬레이션이 현실적으로 작동).
- 입력은 **푸시투토크**(버튼 누를 때만 녹음) = 시끄러운 가게 소음 1차 차단.
- 모든 답변은 **텍스트 + 음성 동시** 제공 (조건 4 충족).

## 3. 곤란질문 → 직원 호출 (에스컬레이션) 케이스

> **원칙**: AI가 추측해서 답하면 안 되는 질문 = 무조건 직원으로 넘긴다. 술집 특성상 **법적 책임·안전·돈** 관련은 전부 에스컬레이션.

| 분류 | 예시 질문 | 처리 |
|---|---|---|
| **A. 안전·법적 책임** | "이거 먹으면 알레르기 괜찮아요?" / "임신했는데 마셔도 돼요?" | AI 단정 금지 → "안전을 위해 직원이 확인해 드릴게요" + **직원 호출** |
| **B. 주류·연령·만취** | 미성년 의심 / 만취 후 추가 주문 / 신분증 분쟁 | 즉시 **직원 호출** (AI가 판매 거부·판단 안 함) |
| **C. 돈·할인·계산 분쟁** | "깎아주세요" / "계산이 이상해요" / 환불 요청 | AI 협상 금지 → **직원 호출** |
| **D. 재고·예약·대량주문** | "단체 20명 돼요?" / "이거 품절?" / "예약할게요" | 실시간 정보 필요 → **직원 호출** (또는 RAG에 재고연동 시 답변) |
| **E. 컴플레인·분실물** | "음식이 이상해요" / "지갑 잃어버렸어요" | 감정·책임 사안 → **직원 호출** |
| **F. 개인정보·외부요청** | 영수증 정보, 사장 연락처, 사적 질문 | **직원 호출** 또는 정중히 거절 |
| **G. 신뢰도 미달 (catch-all)** | FAQ·RAG에 없고 LLM 확신도 낮음 | "정확히 답변드리기 어려워 직원을 불러드릴게요" + **직원 호출** |

**직원 호출 동작 (구현) — 디지털 알림 없음 / 아날로그 방식 (v0.3 확정)**
- 타치노미 특성상 손님이 **손을 들어 직접 직원을 부르는** 1차원적 방식이 자연스럽고 빠르다. 별도 알림 시스템(푸시·n8n·슬랙) **구현 안 함.**
- 에스컬레이션 트리거 시 AI 동작 = **화면+음성으로 "직원을 불러주세요"를 손님 언어로 안내** (예: "For this, please raise your hand to call our staff"). 끝.
- 즉 AI의 역할은 "직원에게 신호 보내기"가 아니라 **"손님에게 직원을 부르라고 자기 언어로 알려주기"**. 부르는 행위 자체는 손님이 몸으로 한다.
- 이점: 서버·알림 인프라 0, 구현 난이도 급감, 매장 현실과 100% 일치.

## 4. 1단 / 2단 응답 구조 설계 (핵심 아키텍처)

### 4.1 전체 흐름

```
[손님 음성]
   │  (푸시투토크 녹음)
   ▼
① STT  음성 → 텍스트 (+ 언어 자동감지: en/zh/ja/ko)
   │
   ▼
② 의도 매칭  ── FAQ 캐시와 대조 (임베딩 유사도)
   │
   ├─ 매칭 O ─────────────▶ [1단·즉답]  미리 저장된 답변 텍스트 + TTS mp3 즉시 재생
   │                          ⚡ ~1초 이하 / LLM·TTS 호출 0 / 비용 0
   │
   └─ 매칭 X ─┐
             ▼
   ③ 민감/곤란 분류 ── 에스컬레이션 트리거(3장 A~G)?
             │
             ├─ YES ───────▶ [직원 호출]  안내 음성 + 직원 알림
             │
             └─ NO ─┐
                    ▼
          ④ [2단·생성]  RAG 검색(메뉴·가게정보) → LLM(가게 페르소나) → TTS
                          🐢 ~3~6초 / 호출당 비용 발생
                    │
                    ▼
               [텍스트 + 음성 응답]
```

### 4.2 1단 (캐시 / 즉답)

- **무엇을**: FAQ 10개 + 자주 나오는 변형 질문.
- **어떻게**: 답변 텍스트뿐 아니라 **TTS 음성파일(mp3)까지 사전 생성**해 저장. 매칭되면 그대로 재생.
- **효과**: 응답 1초 이하, LLM·TTS 런타임 호출 0 → **비용 0, 속도 최상**.
- **매칭 방식**: 손님 질문 임베딩 ↔ FAQ 임베딩 코사인 유사도 ≥ 임계값(예 0.85)이면 히트.

### 4.3 2단 (생성 / RAG + LLM)

- **언제**: FAQ에 없고, 곤란질문도 아닐 때만.
- **무엇을**: 가게 톤·맥락을 system prompt에 주입 → RAG로 메뉴·가게정보 검색 → LLM이 생성형 답변.
- **할루시네이션 차단 (★ 김선국 약점 보완 지점)**:
  - LLM은 **RAG로 검색된 사실 안에서만** 답하도록 강제 (예: "검색된 메뉴 정보에 없으면 추측하지 말고 직원 호출").
  - 가격·재고·알레르기 등 사실 정보는 **2단에서 생성하지 말고 1단/RAG 원본 그대로** 인용.
  - 확신도 낮으면 → 3장 G(직원 호출)로 폴백.

### 4.4 가게 페르소나 (2단 system prompt 뼈대)

```
너는 [가게명], 부산 광안리의 일본풍 타치노미(스탠딩 바)의 AI 직원이다.
```

## 6. [확장 기능] 생맥주 설명 내레이션 다국어화

> **목적**: 푸어링 직원이 생맥주를 따르며 하는 맥주 설명 멘트(한국어)를, 외국인 손님에게도 동일하게 다국어로 제공. ganpai의 차별화 포인트(Sapporo 퍼펙트 푸어링)를 외국인에게도 전달.

### 6.1 3가지 방식 비교 → 결론

| 방식 | 내용 | 평가 |
|---|---|---|
| 1. 실시간 STT→번역→스피커 | 직원 말 → 실시간 번역 음성 출력 | ❌ **오버엔지니어링.** 멘트가 고정인데 매번 실시간 처리. 푸어링 소음 = STT 정확도 붕괴. 지연·비용·오류 |
| 2. 실시간 번역 앱 출력 | 위와 동일(출력만 앱) | ❌ 동일 문제 |
| 3. **사전 녹음 mp3 트리거** | 고정 멘트를 미리 번역·녹음 → 트리거 시 재생 | ✅ **정답.** 무지연·완벽 음질·비용0·소음무관. FAQ 1단 캐시와 동일 원리 |

**결정: 방식 3.** 멘트가 표준 고정 스크립트이므로 미리 영/중/일로 번역 + TTS 녹음해 저장. 재생만 한다.

### 6.2 트리거 방식 (유일하게 남은 결정)

푸어링 직원은 손이 없으므로 버튼 누르기 어려움. 후보:
- **(A) 테이블 앱 버튼** — 손님(또는 옆의 홀 직원)이 "맥주 설명 듣기" 탭. **MVP 최단 구현.** 추천 시작점.
- **(B) 음성 핫워드** — 직원이 특정 문장("설명 시작") 말하면 자동 재생. 손 안 쓰는 장점이나, 소음 환경 오작동·미작동 리스크 → **v2로 미룸.**
- **(C) 물리 트리거** — 발판/손목밴드/팔꿈치 버튼. 하드웨어 추가.

> **권장**: MVP는 (A) 버튼으로 시작 → 검증 후 (B) 핫워드 추가. 언어는 테이블 앱에 이미 설정된 손님 언어로 자동 재생.

### 6.3 내레이션 원본 (한국어) + 영어 번역

**한국어 (원본 / TTS 녹음용 확정 대본)**
> 반갑습니다. 저희 ganpai 광안2호점에서는 맥주에 대한 설명과 함께 맥주를 제공해드립니다. 시작하겠습니다.
> 먼저, 맥주를 따르기 전에 하는 작업을 린싱(rinsing)이라고 합니다. 잔을 깨끗이 세척하고 차갑게 만드는 칠링(chilling) 역할을 합니다.
> 저희 ganpai의 푸어링 기법은 삿포로 본사의 퍼펙트 푸어링 기법입니다. 맥주는 향이 날아가지 않게 45도 기울여 70%까지만 따릅니다. 나머지 30%는 크리미한 거품으로 맥주 헤드(head)를 형성해 풍미를 유지하고 산화를 방지하여, 첫 입부터 마지막 모금까지 일관된 맛을 지켜줍니다.
> 마지막으로 표면의 거친 거품을 깔끔하게 제거하고, 잔을 깨끗이 씻어 완성합니다. 저희 맥주는 지금 바로 한 입 드시는 게 가장 시원하고 맛있습니다. 감사합니다.

**English (translation / EN TTS 녹음용)**

> First, the step before pouring is called "rinsing." It cleans the glass and chills it — what we call "chilling."
> Our pouring method is the "Perfect Pour" technique used at Sapporo's headquarters. We tilt the glass 45 degrees and fill it only to 70%, so the aroma doesn't escape. The remaining 30% forms a creamy foam head that locks in the flavor and prevents oxidation — keeping the taste consistent from the first sip to the last.
> Finally, we skim off the rough foam on top and rinse the glass clean to finish. Our beer is at its coldest and most refreshing the very moment you take your first sip. Enjoy, and thank you!
