# 📱 ganpai PWA 실행 · 모바일 설치 가이드

> 버튼식 키오스크 프론트(`index.html`)를 데스크탑·모바일에서 실행하고, 홈 화면에 앱처럼 설치하는 방법.
> 막힐 때 여기만 보면 된다.

---

## 0. 전제

- **맥과 폰이 같은 Wi-Fi**에 연결돼 있어야 한다. (폰이 LTE/5G로 빠지면 안 됨)
- 그냥 `index.html`을 더블클릭하면 보안정책(`file://`)상 JSON·음성을 못 읽는다. **반드시 로컬 서버로 띄운다.**

---

## 1. 맥에서 실행 (PWA 풀기능: 설치·오프라인 다 됨)

터미널에서:

```bash
cd ~/Main\ Project/ganpai_customer_service_ai_voice
python3 -m http.server 8000
```

→ `Serving HTTP on :: port 8000 ...` 뜨고 커서가 멈춰 대기하면 정상.
**이 터미널 창은 끄지 마라** (끄면 서버도 꺼짐). 끌 때는 그 창에서 `Ctrl + C`.

브라우저에서 → **`http://localhost:8000`**
주소창 옆 **설치 아이콘**으로 앱처럼 설치 가능. (localhost는 https 없이도 PWA 풀기능 동작)

---

## 2. 맥 IP 주소 확인 (폰 접속용)

**새 터미널 창**을 하나 더 열고(기존 서버 창은 그대로 둠):

```bash
ipconfig getifaddr en0
```

→ `192.168.0.12` 같은 숫자가 나옴 = 맥 주소. 적어둔다.
빈 줄이면 Wi-Fi가 `en1`일 수 있음 → `ipconfig getifaddr en1` 로 재시도.

---

## 3. 폰 브라우저에서 접속

1. 폰 브라우저(Safari / Chrome / 삼성 인터넷)를 연다.
2. 맨 위 **주소창(URL 칸)** 을 누른다. ⚠️ 검색창 아님.
3. 2단계에서 나온 IP + `:8000` 을 그대로 입력:
   ```
   192.168.0.12:8000
   ```
   (앞에 `http://`는 자동으로 붙음)
4. **이동 / Go / 엔터**를 누른다.

> ⚠️ 흔한 실수: 숫자만 치고 두면 "검색어"로 인식돼 구글 검색이 뜬다. **이동(엔터)** 까지 눌러야 접속됨.

→ 🏮 ganpai 안내 화면이 뜨고 버튼 누르면 음성이 나오면 성공.

---

## 4. 홈 화면에 아이콘으로 추가 (선택 · 보기용)

> 기능엔 영향 없음. 앱처럼 보이게 하는 장식 단계.

- **아이폰(Safari):** 하단 **공유 버튼(↑네모)** → 스크롤 → **"홈 화면에 추가"** → 추가
- **갤럭시(Chrome):** 우측 상단 **⋮ 점 3개** → **"홈 화면에 추가"**
- **갤럭시(삼성 인터넷):** 하단 **≡ 메뉴** → **"현재 페이지 추가"** → **"홈 화면"**
  - 안 보이면: 갤럭시 스토어에서 삼성 인터넷 업데이트, 또는 그냥 Chrome 사용

---

## 5. http vs https — "앱 설치"가 안 떠도 정상

지금처럼 맥에서 **LAN(`http://192.168...`)** 으로 접속하면:

| 항목 | http LAN (지금) | localhost(맥) | https 배포(상용화) |
|---|---|---|---|
| 화면·음성 재생 | ✅ | ✅ | ✅ |
| 홈 화면 바로가기 추가 | ✅ | ✅ | ✅ |
| 진짜 "앱 설치"(독립 실행) | ❌ | ✅ | ✅ |
| 오프라인 캐싱(서비스워커) | ❌ | ✅ | ✅ |

- 폰 LAN(http)에서 **"앱 설치"가 안 뜨는 건 브라우저 문제가 아니라 http라서다.** Chrome으로 바꿔도 동일.
- **완전한 PWA(설치+오프라인)** 는 **HTTPS 배포** 후에야 폰에서 활성화됨 → 상용화 단계 작업.

---

## 6. 상용화(HTTPS 배포) 시 — 버전 올려 업그레이드

PWA는 파일 묶음이라 데스크탑에서 계속 개발하며 버전업 가능.

1. 데스크탑에서 개발·테스트 (localhost)
2. 콘텐츠·음성 수정했으면 → `faq_cache.json` 고치고 → **mp3 다시 굽기**(`generate_tts.py`)
3. **`sw.js` 의 `CACHE = "ganpai-v1"` → `"v2"` 로 올림** (안 올리면 옛 캐시가 계속 재생됨)
4. HTTPS 호스트(Cloudflare Pages / Vercel / Netlify, 무료)에 배포
5. → 매장 태블릿들이 **다음 실행 때 자동으로 새 버전 수신**

> 네이티브 앱과 달리 **앱스토어 재심사 없이** 푸시 한 번으로 전 매장 즉시 갱신. PWA의 최대 강점.

---

## 7. 안 될 때 체크리스트

1. 맥·폰이 **정말 같은 Wi-Fi** 인가? (폰 Wi-Fi 켜짐 확인)
2. 1단계 **서버 터미널이 켜져 있나?**
3. 주소 끝에 **`:8000`** 붙였나?
4. 그래도 안 되면 맥 **방화벽** 확인: 시스템 설정 → 네트워크 → 방화벽 끄고 재시도.

## 관련 파일

- `index.html` — 키오스크 프론트 (faq_cache.json 읽기 + audio/ 재생)
- `manifest.webmanifest` — 앱 이름·아이콘·전체화면 설정
- `sw.js` — 오프라인 음성 캐싱 (콘텐츠 변경 시 CACHE 버전 ↑)
- `icon-192.png` / `icon-512.png` / `apple-touch-icon.png` — 앱 아이콘
- 음성 다시 굽는 법 → README / `generate_tts.py`
