# [개발일지 #2] 설계만 있던 프로젝트에서, 드디어 첫 음성을 구웠다

> 부산 광안리 타치노미 다국어 AI 음성 응대 시스템의 두 번째 기록.
> 이번 편은 솔직히 "설계서에 글자로만 있던 게 처음으로 소리가 된 날"의 기록이다.

## 머리는 다 굴렸는데, 들리는 결과물은 0이었다

지난 편까지 설계는 A급이었다. 응답 2단 구조, 4개 언어 콘텐츠, 코드/콘텐츠 분리, 데모 파이프라인까지 다 짰다. 그런데 며칠 폴더를 다시 열어보고 깨달았다. **`audio/` 폴더가 텅 비어 있었다.**

이 프로젝트의 심장은 "사전 캐시 즉시 재생"이다. 손님이 물으면 LLM도 TTS도 안 돌리고 미리 구워둔 mp3를 바로 트는 것. 그런데 정작 그 mp3가 한 개도 없었다. 설계서에는 "1단 캐시"라고 백 번 써놨지만, 실증된 건 0%였다.

머리로는 다 끝냈다고 착각하고 있었던 거다. 그래서 이번엔 글이 아니라 **소리**를 만들기로 했다.

## 막혔던 빈칸부터 닫았다

음성을 구우려면 대본에 빈칸이 없어야 한다. `faq_cache.json`에 두 군데가 비어 있었다.

- 화장실 비밀번호: `[TOILET_CODE]`
- 와이파이 정보: `[WIFI_ID/PW]`

이건 외부에서 받을 것도, 누구와 협의할 것도 없는 데이터였다. 내가 현업자라 머릿속에 답이 있었다. 그동안 안 채운 것 자체가 "결정 지연"이었다는 걸 인정했다.

- **화장실 비번**은 실제 값(`2026*`)을 4개 언어 대본에 모두 입력했다.
- **와이파이**는 생각을 바꿨다. AI가 비번을 읽어주는 것보다, "직원이 직접 안내해 드릴게요, 손 들어 불러주세요"로 에스컬레이션 처리하는 게 낫다고 판단했다. 와이파이 비번은 자주 바뀌고, 잘못 읽히면 손님이 헤맨다. 사람이 처리할 건 사람에게 넘기는 게 이 프로젝트의 일관된 원칙이기도 하다.

이렇게 빈칸을 0개로 만들었다. 빈칸이 남은 대본은 스크립트가 자동으로 SKIP하도록 짜놨기 때문에, 이 작업을 끝내야 56개 전부가 생성 대상이 된다.

## 그리고 진짜 벽: `externally-managed-environment`

대본을 다 채우고 `pip install edge-tts`를 쳤더니 빨간 에러가 떴다.

```
error: externally-managed-environment
× This environment is externally managed
```

처음엔 당황했다. 하지만 읽어보니 위험한 게 아니라 **Homebrew Python이 시스템 보호를 위해 전역 설치를 막는 정상 동작**(PEP 668)이었다. 시스템 Python에 아무 패키지나 깔다 깨지는 걸 막으려는 거다.

정석은 **가상환경(venv)**이다. 이 프로젝트 전용 공간에만 패키지를 깔아서 시스템을 안 건드린다.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install edge-tts
python3 generate_tts.py --backend edge
```

`(.venv)`가 프롬프트 앞에 붙은 상태에서 설치하니 에러 없이 통과했다. 끝나고 나올 땐 `deactivate`. 막혔을 때 덮어버리지 않고 에러 메시지를 끝까지 읽은 게 핵심이었다.

## 결과: 56개 음성이 실제로 들렸다

```
총 56개 / 생성 56개 / 보류(빈칸) 0개  [backend=edge]
  ✅ order_ko.mp3
  ✅ order_en.mp3
  ...
  ✅ toilet_wifi_zh.mp3
```

그리고 마지막으로:

```bash
afplay audio/order_ja.mp3
```

스피커에서 일본어 주문 안내가 흘러나왔다. 설계서에 글자로만 박혀 있던 문장이 처음으로 **소리**가 된 순간이었다. FAQ 9종, 에스컬레이션, 맥주 설명, 케그 이벤트(삿포로·에비스 2버전 + 가위바위보)까지 4개 언어 전부.

## 이번에 내 기술이 된 것

- **PEP 668 / venv**: 왜 전역 pip가 막히는지, 가상환경으로 어떻게 우회하는지 이제 손에 익었다. 앞으로 이 프로젝트에서 패키지 설치로 막힐 일은 없다.
- **빈칸 SKIP 로직 검증**: dryrun으로 먼저 "생성 56 / 보류 0"을 확인하고 실제로 구웠다. 음성 굽기 전에 로직부터 점검하는 습관.
- **운영 판단**: 와이파이를 AI가 읽을지 직원이 안내할지 — 기술이 아니라 손님 경험 기준으로 결정했다.
