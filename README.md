# 🍺 다국어 AI 음성 응대 시스템 — ganpai 광안2호점 MVP

> 부산 광안리 일본풍 타치노미(스탠딩 바)를 위한 **다국어 음성 Q&A + 이벤트 참여** AI 시스템.
> 외국인 손님(전체의 40%)이 자기 언어(영·일·중)로 묻고, 가게의 정보·문화·이벤트를 실시간으로 즐길 수 있게 한다.

---

## 1. 문제 (Problem)

광안리 해수욕장 근처, 외국인 손님 비중 **40%**(영어·중국어·일본어).
- 푸어링 직원은 생맥주를 따르느라 손이 없어 외국어 응대가 불가능하다.
- 홀 직원은 손이 있어도 3개 국어를 능숙하게 응대하기 어렵다.
- 가게만의 **맥주 설명**과 **케그 교체 이벤트**가 있지만, 외국인은 무슨 일이 벌어지는지 이해하지 못해 **참여하지 못한다.**

→ 매일 외국인 매출·경험·재방문 기회를 흘리고 있다.

## 2. 해결 (Solution)

테이블 NFC/태블릿에서 동작하는 **손님 셀프형 AI 음성 도우미**.
- 손님이 자기 언어로 음성 질문 → AI가 **텍스트 + 음성**으로 같은 언어로 응답.
- 자주 묻는 질문은 즉답, 곤란한 질문은 직원 호출 안내, 가게 이벤트는 다국어로 참여 유도.

핵심 가치는 직원 노동 대체가 아니라 **언어 장벽 해소 + 문화 경험 전달**이다.

---

## 3. 핵심 설계 (Architecture)

### 3.1 2단 응답 구조

```
[손님 음성]
   │  STT (음성 → 텍스트 + 언어감지)
   ▼
의도 매칭 ──┬─ 매칭 O ───────────▶ [1단·즉답]  사전 생성 답변 + mp3 즉시 재생
            │                        ⚡ ~1초 / LLM·TTS 호출 0 / 비용 0
            ├─ 곤란/민감 질문 ──────▶ [직원 호출]  "손 들어 직원을 불러주세요" 안내
            └─ 매칭 X ─────────────▶ [2단·생성]  RAG + LLM (검색된 사실 안에서만)
```

**설계 포인트**
- **사전 캐시(1단)가 핵심**: 자주 나오는 질문·고정 멘트는 미리 번역·녹음(mp3)해 두고 즉시 재생.
  실시간 STT→번역→TTS 대비 **무지연·완벽 음질·비용 0·소음 무관**.
- **할루시네이션 차단**: 2단 LLM은 RAG로 검색된 사실 안에서만 답하고, 모르면 직원 호출로 폴백.
- **에스컬레이션은 아날로그**: 타치노미 특성상 손님이 손 들어 직접 부르는 방식. 별도 알림 인프라 불필요.

### 3.2 콘텐츠 모듈

| 모듈 | 내용 | 트리거 |
|---|---|---|
| FAQ Q&A | 주문·추천·자릿세·결제·영업시간·화장실/와이파이·흡연·매운맛·무알콜 (9종 × 4개 언어) | 음성 질문 |
| 맥주 설명 내레이션 | Sapporo 퍼펙트 푸어링 설명 (4개 언어) | 버튼 탭 |
| 케그 교체 이벤트 | ① 종 안내(맥주 2종) → ② 가위바위보(우승 시 삿포로 한 잔) (4개 언어) | 직원/버튼 |

## 4. 기술 스택

| 영역 | 사용 기술 |
|---|---|
| 모바일 | (예정) Flutter / React Native — 태블릿·모바일 |
| STT | Whisper / Google STT (다국어·언어자동감지) |
| TTS | edge-tts(뉴럴) / macOS `say`(오프라인) — 사전 mp3 생성 |
| 벡터DB(RAG) | Supabase pgvector |
| LLM | Claude / GPT (2단 생성, RAG 근거 강제) |
| 데이터 | 단일 `faq_cache.json` (콘텐츠·코드 분리) |

## 5. 저장소 구조

```
.
├── faq_cache.json      # 모든 콘텐츠(4개 언어 FAQ·에스컬레이션·내레이션·케그이벤트). 코드와 분리.
├── demo_faq_bot.py     # 응답 파이프라인 데모 (1단/2단/에스컬레이션 + 내레이션·케그 재생)
├── generate_tts.py     # faq_cache.json → audio/ mp3 자동 생성 (edge-tts / macOS say)
├── audio/              # 생성된 음성 캐시 (52개, generate_tts.py 실행 후)
└── README.md
```

> **콘텐츠와 코드를 분리**한 게 핵심. 메뉴·가격·멘트가 바뀌어도 코드는 안 건드리고 `faq_cache.json`만 수정 → 재생성.

## 6. 실행 방법

```bash
# 1) 응답 로직 데모 (API 키 불필요, 즉시 실행)
python3 demo_faq_bot.py            # 예시 질문 + 내레이션 + 케그 이벤트 자동 시연
python3 demo_faq_bot.py --chat     # 직접 타이핑해서 대화

# 2) 음성 캐시(mp3) 굽기  ※ 맥에서 실행 권장
pip install edge-tts
python3 generate_tts.py --backend edge      # 고품질 뉴럴 음성(.mp3)
# 또는 설치 없이 macOS 내장 TTS:
python3 generate_tts.py --backend say       # 오프라인(.aiff)

# 3) 음성 재생 데모 (audio/ 에 mp3가 있으면 실제 소리 재생)
python3 demo_faq_bot.py --narration ja
python3 demo_faq_bot.py --keg yebisu --lang zh
```
